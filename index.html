入門

FFG IIoT數據服務平台介紹

友嘉工業物聯網數據服務平台 (FFG IIOT Data Service Platform)，提供數據採集和資料分析的服務。

FFG IIoT數據服務平台是PaaS，負責通過工廠端網關的微服務框架進行數據獲取或任務分配。在平台上，可以通過不同的域來管理位於不同工廠位置的設備。可以為AI收集設備數據機器學習引擎。另外，規則引擎負責檢查報告的事件，並根據規則觸發警報通知。

為符合多租戶的需求，平台將各廠區，各專案劃分為不同的網域(domain)來進行管理。每個網域下面包含獨立的網關(gateway)、設備(device)和資料(data)。

管理員可同時創建多個網域以及新增、刪除使用者。而使用者擁有網關和設備的管理權限，網關負責把設備群的數據或事件採集後上報，讓使用者可監看資料。

AI工廠

為用戶提供一個簡單的過程來創建數據集並建立用於聚類，分類，回歸和預測的特定用途的AI模型。一個完整的模性創建過程會經過以下的步驟:

  1. 建立資料集 Dataset
  2. 建立模眝 Bucket
  3. 建立前處理程序
  4. 創建模型
  5. 觀看訓練結果
  6. 發佈任務 Mission

-模眝 Bucket

Bucketu由以下元素所構成

  • 前處理程序 preprocessing
  • 模型 model
  • 訓練結果 training result

點擊 Add 新建Bucket，並填入以下資料

  • 模型的種類 Model Category
  • 資料集 Dataset
  • 訓練目標 Target

點擊View Table統計資料Detail和原始資料Data

顯示Bucket列表

MANAGEMENT功能 說明
DETAIL bucket詳情
EDIT 編輯bucket
DELETE 刪除bucket

點擊Detail可以進入功能頁面，頁面上面顯示當前Bucket資訊:

  • 前處理 Preprocessing
  • 模型 Model
  • 訓練結果 Training Result

模眝 Bucket - 前處理 Preprocessing

點擊preprocessing顯示前處理列表
view可以觀看前處理流程和結果
edit編輯前處理流程，當model有用到時呈現灰色，表示不可編輯

建立preprocess:點擊Add，輸入前處理名稱

Preprocess 編輯頁面:

主要由三部分組成
A 前處理指令列
B 前處理編輯視窗
C 前處理結果

左側A為前處理指令列表分為以下類別，點擊內部指令+加入前處理視窗。

  • 清理 Cleaning
種類 說明
drop column 將選擇的column捨棄
drop row 將選擇的row捨棄
fillna 將選擇的column中含有空值(None或Null)進行取代，可選擇不同取代方式
rename 將選擇的column進行重新命名
  • 整合 Integration
種類 說明
merge 將不同總類的column之值相加
  • 轉換 Transformation
種類 說明
normalize 將選擇的column之值進行正規化
standardize 將選擇的column之值進行標準化
minmax 將選擇的column之值進行歸一化 (將數值資料轉化至0到1區間)
robust 將選擇的column之值進行強健化 (針對數值異常值進行處理)
label encoding 將選擇的column之值進行標籤編碼 (將類別型資料轉換成數值，例如:0,1,2…)
label binary 將選擇的column之值進行標籤二值化 (將類別型資料轉換成0或1)
one-hot encoding 將選擇的column之值進行獨熱編碼 (將類別型資料拆成多個column，每個column中的數值由1或0代替)
ordinal encoding 將選擇的column之值進行序號編碼 (將類別型資料轉換成數值，特別用於有大小或是次序關係的資料)
tfidf vectorizer 詞頻與逆向文件頻率分析，將文字集中的各個詞轉換成各個column並以數值表示其詞頻分析
  • 壓縮 Reduction
種類 說明
pca 主成分分析
incremental pca 增量主成分分析，適用於data量太大的時候，採取分批處理
kernel pca 核主成分分析，將非線性可分的資料轉換至高為空間中再使用pca
truncated svd 截斷奇異值分離，適用於數據矩陣並進行降維
  • 其它 Other
種類 說明
copy 將選擇的column複製一個新的column
code 自行coding

中間Flow B 為前處理編輯視窗:分為特徵(Feature)和目標(target)
可將前處理指令

  • 加入:拖拉指令至flow視窗內
  • 刪除:拖拉指令至flow視窗外
  • 插入:拖拉交換位置

右側結果列表:

  • 原始資料Data
  • 統計資料Detail
  • 輪廓資料Profile
說明 指令
編輯指令 EDIT
刪除指令 DELETE

特徵 Feature 編輯視窗

目標 target 編輯視窗

編輯指令參數: 新增指令後需要填入參數,右側的結果列表會即時更新

Data 原始資料:
當點擊原始資料Data的表頭欄位時有用到的前處理指令會顯現黃色

Detail 統計資料:
描述資料的概況包含:數量、型別、平均值、標準差、四分衛數

Profile 輪廓資料:
初次建立的時間依據數據的大小而定,當資料要更動時需要按下Update Profile來更新

載入前處理程序(Loading):
列出相同的dataset的其它前處理列表,載入後即可套用到此前處理程序

模眝 Bucket - 模型 Model

點擊Model顯示Model列表

MANAGEMENT功能 說明
EDIT model 名稱編輯
TUNE 載入原先model參數並創立新model
DELETE 刪除model

點擊Add新增Model
一共有三個程序，依序執行才算完成model建立

  1. 資料繫結 Sink
  2. 資料前處理 Preprocess
  3. 模型建立 Make

Sink: 輸入model 的名稱和描述
按下preview可以預覽Dataset的Data和Detail

Proprecess:

左邊視窗:選擇前處理的程序，Raw Data 表示套用原始資料
右邊視窗:前處理後的結果呈現 Data、Detail、Profile

點擊VIEW FEATURES或VIEW TARGETS 可以顯示相對應的結果
Profile圖表將 features 和 targets 資訊綜合在一起。

Data 原始資料

Detail 統計資料

Profile 輪廓資料

Make 需要輸入參數模型，才能開始訓練

  • Basic
    • input :模型輸入
    • model type: 模型類型 (依據Model Category 有所不同)
    • output: 模型輸出
  • Split Data
    • testing data size:測試集佔整體資料的比例
    • Random state:決定資料隨機的方式
    • Shuffle:拆分前是否先將資料打亂
  • Training Parameter: 有以下參數類型 (依據model type有所不同)
    • string
    • boolean
    • number

Model category 和 Model type 列表
| Regression | Classification | Clustering |
| ————- | ————– | ———- |
| SGD | SGD | KMeasn |
| Linear SVR | Linear SVC | GMM |
| Lasso | KNeighbors | MeanShift |
| Decision Tree | DecisionTree | |
| RandonForest | RandomForest | |
| AdaBoost | AdaBoost | |

輸入完成後點擊Training，開始訓練模型

訓練時間依據模型的複雜度有所不同，完成後右上角會跳提醒。
可以到Training Result查看訓練結果。

模眝 Bucket - 訓練結果 Training Results

點擊Training Result顯示該bucket所有model訓練的結果並進行分析比較
| MANAGEMENT功能 | 說明 |
| ————– |:———— |
| REPORT | 訓練結果報告 |
| DELETE | 刪除model |

REPORT:顯示Classification 中 training& testing 所有資料的 訓練結果

Comparison Results: 模型結果比較，需要輸入以下參數
Data Type可以選擇

  • Training Comparison
  • Testing Comparison
  • Testing VS Training

Metric:要比較的項目

Chart Type:

  • SINGLE : 每個Metric用一張圖來呈現
  • MIX : 多個Metric呈現在同一張圖上

Data Type = Training Comparison / Testing Comparison

  • Chart Type = SINGLE

  • Chart Type = MAX

Data Type = Testing VS Training

勾選要比較的模型(單選)

  • Chart Type = SINGLE

  • Chart Type = MAX

-任務 Mission

當模型建立完成後即可透過 API 來進行預測 Predict
點擊 Mission 顯示當前的 Mission 列表

MANAGEMENT功能 說明
DETAIL Mission 細節 (Prediction、History)
EDIT Mission 名稱編輯
DELETE 刪除Mission

點擊Detail，視窗上半部顯示該Mission的相關資訊

  • Feature : 特徵值
  • Target: 目標值
  • Post API Array: 使用陣列呼叫 API
  • Mission Array Probability: 回傳機率陣列
  • Post API file: 使用 .txt 呼叫API

視窗下半部顯示:

  • Prediction: 提供線上即時預測的範例和測試
  • History: 呼叫 API 的結果

預測 Prediction
輸入特徵值 featrue 取得 target 預測的結果

點擊 Edit 輸入相關參數，若沒有輸入回傳預設值為null

點擊 Predict 開始預測，結果會顯示在Action右側欄位

History
載入預測 API 呼叫列表，顯示呼叫時間和狀態 STATUS
